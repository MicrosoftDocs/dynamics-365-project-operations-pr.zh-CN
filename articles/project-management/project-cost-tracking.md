# <a name="labor-cost-tracking-on-projects"></a><span data-ttu-id="b4624-103">项目人工成本跟踪</span><span class="sxs-lookup"><span data-stu-id="b4624-103">Labor cost tracking on projects</span></span>

<span data-ttu-id="b4624-104">_**适用于：** 基于资源/非库存场景的 Project Operations，精简部署 - 估价开票交易_</span><span class="sxs-lookup"><span data-stu-id="b4624-104">_**Applies To:** Project Operations for resource/non-stocked based scenarios, Lite deployment - deal to proforma invoicing_</span></span>

<span data-ttu-id="b4624-105">Dynamics 365 Project Operations 可以针对项目规划按最低所需粒度跟踪人工估算值和支出。</span><span class="sxs-lookup"><span data-stu-id="b4624-105">Dynamics 365 Project Operations tracks labor estimates and spend at the lowest required granularity on the project plan.</span></span> <span data-ttu-id="b4624-106">人工成本财务估算依据的是分派给项目规划中的每个叶节点任务的计划工作以及通用资源或指定资源。</span><span class="sxs-lookup"><span data-stu-id="b4624-106">The financial estimate of labor cost is based on the planned effort and the generic or named resources that are assigned to each leaf-node task in the project plan.</span></span> <span data-ttu-id="b4624-107">当项目开始并且人们开始报告各项项目任务的时间时，将对实际人工支出进行汇总，从而开始进行预测计算。</span><span class="sxs-lookup"><span data-stu-id="b4624-107">When the project begins and people start to report time on various project tasks, the actual spend on labor is summarized which starts a calculation of the projections.</span></span>

## <a name="labor-cost-tracking-view"></a><span data-ttu-id="b4624-108">人工成本跟踪视图</span><span class="sxs-lookup"><span data-stu-id="b4624-108">Labor Cost Tracking view</span></span>

<span data-ttu-id="b4624-109">在 **项目** 页上的 **跟踪** 选项卡上，可以选择 **跟踪** > **成本** 来打开 **成本跟踪** 视图，并查看项目规划中每个任务所花费的人工的进度。</span><span class="sxs-lookup"><span data-stu-id="b4624-109">On the **Projects** page, on the **Tracking** tab, you can select **Tracking** > **Cost** to open the **Cost Tracking** view and see the progress of labor spend on each task in the project plan.</span></span> <span data-ttu-id="b4624-110">此视图还将任务所花费的实际人工成本与任务的计划人工成本进行比较。</span><span class="sxs-lookup"><span data-stu-id="b4624-110">This view also compares the actual labor cost spent on a task to the task's planned labor cost.</span></span> <span data-ttu-id="b4624-111">Project Operations 使用以下公式计算人工成本指标：</span><span class="sxs-lookup"><span data-stu-id="b4624-111">Project Operations uses the following formulas to calculate labor cost metrics:</span></span>

- <span data-ttu-id="b4624-112">**计划成本**：每个叶节点任务中所有资源分派的估算成本</span><span class="sxs-lookup"><span data-stu-id="b4624-112">**Planned cost**: Estimated cost of all resource assignments on each leaf node task</span></span>
- <span data-ttu-id="b4624-113">**实际成本**：针对任务记录的所有时间成本实际值的总和</span><span class="sxs-lookup"><span data-stu-id="b4624-113">**Actual Cost**: Sum of all cost actuals for time recorded on the task</span></span>
- <span data-ttu-id="b4624-114">**成本消耗百分比**：实际成本 ÷ 成本完工估算</span><span class="sxs-lookup"><span data-stu-id="b4624-114">**Cost consumption percentage**: Actual cost ÷ Cost estimate at complete</span></span>
- <span data-ttu-id="b4624-115">**剩余成本**：成本完工估算 - 实际成本</span><span class="sxs-lookup"><span data-stu-id="b4624-115">**Remaining Cost**: Cost estimate at complete  – Actual cost</span></span>
- <span data-ttu-id="b4624-116">**完工成本**：剩余成本 + 实际成本</span><span class="sxs-lookup"><span data-stu-id="b4624-116">**Cost at Complete**: Remaining cost + Actual cost</span></span>
- <span data-ttu-id="b4624-117">**成本差异**：计划成本 - 成本完工估算</span><span class="sxs-lookup"><span data-stu-id="b4624-117">**Cost variance**: Planned cost – Cost estimate at complete</span></span>

<span data-ttu-id="b4624-118">每项任务都会显示任务的成本偏差预估。</span><span class="sxs-lookup"><span data-stu-id="b4624-118">Each task shows a projection of the cost variance on the task.</span></span> <span data-ttu-id="b4624-119">如果成本完工估算超过计划成本，则预计任务会超出预算。</span><span class="sxs-lookup"><span data-stu-id="b4624-119">If the cost estimate at complete is more than the planned cost, the task is projected to go over budget.</span></span> <span data-ttu-id="b4624-120">如果成本完工估算低于计划成本，则预计任务会低于预算完成。</span><span class="sxs-lookup"><span data-stu-id="b4624-120">If the cost estimate at complete is less than the planned cost, the task is projected to finish under budget.</span></span>

>[!NOTE]
> <span data-ttu-id="b4624-121">Project Operations 只在 **项目** 页上的 **跟踪** 选项卡上显示人工成本。尽管可估算和跟踪消耗的材料和支出，但是这些成本未包含在 **跟踪** 选项卡上显示的成本中。此选项卡只适用于通过重新预测人工来重新预测人工成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-121">Project Operations only shows labor costs on the **Project** page on the **Tracking** tab. This tab is designed to work only for reprojecting labor costs by reprojecting effort.</span></span>
<span data-ttu-id="b4624-122">显示的所有成本金额将从用于确定成本费率的项目成本价格货币转换为项目成本货币。</span><span class="sxs-lookup"><span data-stu-id="b4624-122">All cost amounts shown are converted to the project's cost currency from the project's cost price currency used to determine the cost rate.</span></span> <span data-ttu-id="b4624-123">项目的成本货币是项目的合同单位的货币。</span><span class="sxs-lookup"><span data-stu-id="b4624-123">The cost currency of the project is the currency of the contracting unit on the project.</span></span> <span data-ttu-id="b4624-124">**项目** 页面内 **估算** 选项卡上显示的时间的估算成本值可能不会合计达到 **跟踪** 选项卡上的计划成本。之所以出现这种差异，是因为 **估算** 网格上汇总估计成本的方法与 **跟踪** 网格上计算计划成本的方法不同。</span><span class="sxs-lookup"><span data-stu-id="b4624-124">The estimated cost values for time that are shown on the **Estimates** tab on the **Project** page might not add up to the planned cost on the **Tracking** tab. The reason for this discrepancy is because of the differences in how estimated cost is summarized on the **Estimates** grid and how planned cost is calculated on the **Tracking** grid.</span></span> 
>
> - <span data-ttu-id="b4624-125">**估算** 选项卡使用价目表中成本费率的相同货币来计算估计成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-125">The **Estimates tab** calculates the estimated cost by using the same currency of the cost rate in the price list.</span></span> <span data-ttu-id="b4624-126">然后，以价目表货币表示的估算成本会转换为以项目成本货币表示的估算成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-126">Then, the estimated cost in the price list currency is converted to the estimated cost in the project's cost currency.</span></span> <span data-ttu-id="b4624-127">系统会显示四舍五入到两个小数的以项目货币表示的估算成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-127">The estimated cost in the project currency is shown rounded to 2 decimal places.</span></span> <span data-ttu-id="b4624-128">在此转换过程中，不会应用货币精度。</span><span class="sxs-lookup"><span data-stu-id="b4624-128">At no point during this conversion is currency precision applied.</span></span> 
> - <span data-ttu-id="b4624-129">在 **跟踪** 选项卡上，计划成本计算按照略微不同的计算顺序进行，该计算顺序涉及在两个阶段中应用的货币精度：</span><span class="sxs-lookup"><span data-stu-id="b4624-129">On the **Tracking** tab, the planned cost calculation follows a slightly different calculation order that involves currency precision being applied in two stages:</span></span> 
   ><ol>
   ><li><span data-ttu-id="b4624-130">以价目表货币表示的估算成本金额将转换为基础货币（转换 1）。</span><span class="sxs-lookup"><span data-stu-id="b4624-130">The estimated cost amount in the price list currency is converted to the base currency (conversion 1).</span></span></li>
   ><li><span data-ttu-id="b4624-131">以基本货币表示的估算成本金额将转换为项目成本货币（转换 2）。</span><span class="sxs-lookup"><span data-stu-id="b4624-131">The estimated cost amount in the base currency is converted to the project cost currency (conversion 2).</span></span> </li>
   ></ol>
   ><span data-ttu-id="b4624-132">在这两个步骤中都应用了货币精度，以便（在 **跟踪** 选项卡上）获取计划成本，该成本与估算成本（在 **估算** 选项卡上的 **分时段** 视图上）略有不同。</span><span class="sxs-lookup"><span data-stu-id="b4624-132">Currency precision is applied in both steps to get a planned cost (on the **Tracking** tab) that deviates slightly from the estimated cost (on the **Time - phased** view on the **Estimates** tab).</span></span> 
   
## <a name="reprojecting-costs-on-leaf-node-tasks"></a><span data-ttu-id="b4624-133">重新预测叶节点任务的成本</span><span class="sxs-lookup"><span data-stu-id="b4624-133">Reprojecting costs on leaf node tasks</span></span>

<span data-ttu-id="b4624-134">不能在 **项目** 页面 **跟踪** 选项卡上直接重新预测叶节点任务的人工成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-134">Labor costs on a leaf node task can't be directly reprojected on the **Tracking** tab on the **Project page**.</span></span> <span data-ttu-id="b4624-135">但是，您可以使用 **人工跟踪** 视图重新预测任务的剩余工作。</span><span class="sxs-lookup"><span data-stu-id="b4624-135">However, you can use the **Effort Tracking** view to reproject the remaining effort on a task.</span></span> <span data-ttu-id="b4624-136">这会导致重新计算任务的剩余成本。</span><span class="sxs-lookup"><span data-stu-id="b4624-136">This causes a recalculation of the remaining cost on the task.</span></span> <span data-ttu-id="b4624-137">下面说明了其工作原理。</span><span class="sxs-lookup"><span data-stu-id="b4624-137">The following is a description of how this works.</span></span>

1. <span data-ttu-id="b4624-138">项目经理可以用新估算的任务剩余工作量来更新 **剩余工作量** 字段中的默认值，以重新预测任务工作量。</span><span class="sxs-lookup"><span data-stu-id="b4624-138">A project manager can reproject effort on tasks by updating the default value in the **Remaining Effort** field with a new estimate of the remaining effort on the task.</span></span> <span data-ttu-id="b4624-139">重新预测将导致重新计算任务的工作量完工估算 (EAC)、进度百分比和任务的预测工作量差异。</span><span class="sxs-lookup"><span data-stu-id="b4624-139">Reprojecting causes a recalculation of the task's effort estimate at complete (EAC), progress percentage, and the projected effort variance on a task.</span></span> <span data-ttu-id="b4624-140">还会重新计算汇总任务的 EAC、待完成估算 (ETC) 和进度百分比，并生成新的工作量偏差预测。</span><span class="sxs-lookup"><span data-stu-id="b4624-140">The EAC, estimate to complete (ETC), and progress percentage on the summary tasks are also recalculated and produce a new projection of effort variance.</span></span>

4. <span data-ttu-id="b4624-146">系统会重新计算直到根节点的摘要任务的完工成本值。</span><span class="sxs-lookup"><span data-stu-id="b4624-146">The cost at complete value of the summary tasks all the way to the root node are recalculated.</span></span>
