# <a name="configure-accounting-for-billable-projects"></a>配置收费项目会计

_**适用于：** 基于资源/非库存场景的 Project Operations，精简部署 - 估价开票交易_

Dynamics 365 Project Operations 支持收费项目（包括时间和材料以及固定价格交易）的各种会计选项。

- **时间和材料交易**：随着工作的进行，这些交易根据项目的时间、支出、项目或费用的消耗开票。 这些交易成本可以与每笔交易的收入相匹配，项目随着工作的进展开票。 项目收入也可以在交易发生时计入应计。 在开票过程中，会确认收入并冲销应计收入（如果适用）。
- **固定价格交易**：这些交易根据基于项目合同的记帐计划开票。 固定价格交易的收入可以按照 **已完成合同** 或 **完成百分比** 方法在开票时确认或定期计算和过帐。

如果一个项目与一个或多个合同子项关联，该项目被视为可计费。 项目合同子项为其自身定义允许的计费方法和交易类型。

例如，Fabrikam Robotics 赢得了与 Adatum 公司的设备优化项目合同。 Adatum 将支付固定的 10,000 美元金额来支付产生的项目支出。 这是一种固定价格计费方法。 项目时间和费用将按每次使用计费。 这是一种时间和材料计费方法。 所有工作将在名为 Adatum 设备优化的单个项目下进行跟踪。

一个项目团队成员在 Adatum 设备优化项目中提交了八个小时的工作。 当项目经理批准此时间时，系统将使用时间和材料计费方法创建实际交易、发票和客户。 此交易将不包含在固定价格收入预估计算中。

另一个项目团队成员针对 Adatum 设备优化项目提交了 2000.00 美元的旅行支出。 当项目经理批准此提交时，系统将使用固定价格计费方法来创建实际交易和此项目支出的客户。 不会基于此交易为客户开票。 而是将使用单独配置的计费里程碑来开票。 此支出交易以及支出预估将包含在固定价格收入预估计算中。

项目交易的会计原则在项目成本和收入模板中定义。 对于每个项目交易，系统都会使用已配置的项目成本和收入模板规则来确定适当的项目成本和收入模板。

## <a name="define-project-cost-and-revenue-profiles"></a>定义项目成本和收入模板 

项目成本和收入模板确定项目交易的会计规则。 这些模板在项目管理和会计中配置。 

完成以下步骤创建新的项目成本和收入模板。 

1. 转到 **项目管理和会计** > **设置** > **过帐** > **项目成本和收入模板**。 
2. 选择 **新建** 创建新的项目成本和收入模板。
3. 在 **名称** 字段中，输入模板的名称和简要说明。
4. 在 **计费方法** 字段中，选择 **时间和材料** 或 **固定价格**。
5. 展开 **分类帐** 快速选项卡。 此选项卡上的字段定义会计原则，在使用 Project Operations 集成日记帐将项目交易记入日记帐，然后通过“项目发票方案”开票时使用此会计原则。
6. 在 **分类帐** 快速选项卡上的以下字段中选择适当的信息：

    - **过帐成本 – 小时**：

       - *无分类帐*：过帐 Project Operations 集成日记帐时，时间交易成本不会过帐到分类帐。 但是，会计可以在以后使用“过帐成本”功能过帐成本。
       - **余额**：在“分类帐过帐”设置中，时间交易成本将借记到分类帐科目类型 *WIP - 成本值*，并计入 *工资分配科目*。 会计将使用“过帐成本”功能定期将此成本从“余额”科目转移到“损益”科目。
       - **损益**：过帐 Project Operations 集成日记帐时，时间交易成本将借记到分类帐科目类型 *成本*，并计入 **分类帐过帐设置** 页的 **成本** 选项卡上定义的 *工资分配科目*（**项目管理和会计** \> **设置** \> **过帐** \> **分类帐过帐设置**）。 这是时间和材料交易的最常见设置。
        - *永不过帐到分类帐*：时间交易的成本永远不会过帐到分类帐。

    - **过帐成本 – 支出**：

         - **余额**：过帐 Project Operations 集成日记帐时，支出交易成本将借记到 **分类帐过帐设置** 页的 **成本** 选项卡上定义的分类帐科目类型 *WIP - 成本值*，并计入日记帐行上的抵销科目。 支出的默认抵销科目在 **项目管理和会计** > **设置** \> **过帐** \> **支出的默认抵销科目** 中定义。 会计将使用 **过帐成本** 功能定期将此成本从余额科目转移到损益科目。
        - **损益**：过帐 Project Operations 集成日记帐时，支出交易成本将借记到 **分类帐过帐设置** 页的 **成本** 选项卡上定义的分类帐科目类型 *成本*，并计入日记帐行上的抵销科目。 支出的默认抵销科目在 **项目管理和会计** \> **设置** \> **过帐** \> **支出的默认抵销科目** 中定义。
       
    - **分期付款开票**：

        - **余额**：过帐项目发票方案时，分期付款交易（计费里程碑）将计入 **分类帐过帐设置** 页的 **收入** 选项卡上定义的分类帐科目类型 *WIP 已开票 - 分期付款*，并借记到客户余额科目。
         - **损益**：过帐项目发票方案时，分期付款交易（计费里程碑）将计入 **分类帐过帐设置** 页的 **收入** 选项卡上定义的分类帐科目类型 *已开票收入 - 分期付款*，并借记到客户余额科目。 客户余额科目在 **应收帐款** \> **设置** \> **客户过帐模板** 中定义。

   在定义时间和材料计费方法的过帐模板时，您可以选择按每个交易类型（时间、支出和费用）将收入计入应计。 如果将 **应计收入** 选项设置为 **是**，Project Operations 集成日记帐中未计费的销售交易将记录到总账中。 销售值将借记到 **WIP - 销售值科目**，并计入 **分类帐过帐设置** 页上在 **收入** 选项卡上设置的 **应计收入 - 销售值** 科目。 
  
  > [!NOTE]
  > **应计收入** 选项仅在相应的交易类型 **成本** 过帐到损益科目时可用。
    
7. 展开 **预估** 快速选项卡。 此选项卡上的字段定义固定价格收入预估的计算设置。 此选项卡上的字段仅适用于采用 **固定价格** 计费方法的项目成本和收入模板。
8. 在 **预估** 快速选项卡上的以下字段中选择适当的信息：

    - **用于项目完成计算的原则**：

        - **已完成合同**：直到项目结束才进行成本匹配和收入确认。 在项目完成之前，成本在余额中显示为 WIP。
        - **完成百分比**：根据项目完成百分比在每个期间计算应计收入，并过帐到分类帐。 有多种方法可用于计算完成百分比。 这些方法可以基于配置自动执行，也可以手动执行。
        - **无 WIP**：此设置用于较短时间且发票和成本在同一期间发生的固定价格项目。 在此情况下，**分类帐** 快速选项卡上的 **分期付款开票** 字段值会自动设置为 **损益**，以确保在开票时确认收入。 收入预估流程将不会用于此项目成本和收入模板。

    - **匹配原则**：此字段确定如何将计算出的销售值（应计收入）过帐到分类帐。

        - 使用 **销售值** 原则，系统将通过匹配成本和收入，然后将其作为单个金额过帐来计算销售值。
        - 使用 **生产和利润** 原则，系统会将销售值拆分为实现成本和计算利润。 这些都单独过帐。

    - **成本模板**：允许根据交易类型和项目类别对项目交易进行分组，并为这些组定义完成百分比计算规则。
    - **期间代码**：定义为给定的项目成本和收入模板计算收入预估的频率。
    - **用于预估的类别**：用于将销售值（应计收入）过帐到项目交易。 首先，为 **费用** 交易类型配置专用项目类别，然后为此项目类别设置标志 **预估**。 接下来，根据选择的匹配原则，在项目成本和收入模板的 **销售** 值或 **利润** 字段中选择此项目类别。

## <a name="configure-project-cost-and-revenue-profile-rules"></a>配置项目成本和收入模板规则

项目成本和收入模板规则确定在处理任何可计费项目交易时必须使用的项目成本和收入模板。 通过转到 **项目管理和会计** \> **设置** \> **过帐** \> **项目成本和收入模板规则** 来定义规则。

规则可以按项目合同、项目组或特定项目定义。 系统始终会首先选择最高粒度的规则。
