# <a name="extending-time-entries"></a>扩展时间条目

_**适用于：** 基于资源/非库存场景的 Project Operations，精简部署 - 估价开票交易_

Dynamics 365 Project Operations 包括可扩展的时间条目自定义控件。 此控件包括以下功能：

- 在一周内横向输入时间
- 按日、行或周汇总
- 复制行或周
- 使用 HH:mm 或 HH.hh 的时间条目（自动转换为 HH.hh）
- 从工作、预订或约会导入

可以在两个方面扩展时间条目：
- [添加自定义时间条目供您自己使用](#add)
- [自定义每周时间条目控件](#customize)

## <a name="add-custom-time-entries-for-your-own-use"></a><a name="add"></a>添加自定义时间条目供您自己使用

时间条目是在多个场景中使用的核心实体。 在 2020 年 4 月第 1 波中，引入了 TESA 核心解决方案。 TESA 提供 **设置** 实体和新的 **时间条目用户** 安全角色。 还提供了新字段 **msdyn_start** 和 **msdyn_end**，它们与 **msdyn_duration** 有直接关系。 新实体、安全角色和字段使跨多个产品的时间分配方法更加统一。


### <a name="time-source-entity"></a>时间源实体
| 字段 | 描述 | 
|-------|------------|
| 客户  | 创建时间条目时用作选择值的时间源条目的名称。 |
| 默认时间源 [时间源：isdefault] | 默认情况下，默认时间源只能标记一个时间源。 这允许条目默认为时间源（如果未指定）。 |
|时间源类型 [时间源：sourcetype] | 源类型是一个选项（时间条目源类型），允许时间源与应用关联。 Microsoft 会保留大于 190,000,000 的值。|


### <a name="time-entries-and-the-time-source-entity"></a>时间条目和时间源实体
每个时间条目都与一个时间源记录相关联。 此记录确定哪些应用程序应处理时间条目以及如何处理。

时间条目始终是一个连续的时段，连接了开始、结束和持续时间。

逻辑将在以下情况下自动更新时间条目记录：

- 如果提供了以下三个字段中的两个，将自动计算第三个： 

    - **msdyn_start**
    - **msdyn_end**
    - **msdyn_duration**

- **msdyn_start** 和 **msdyn_end** 字段可识别时区。
- 创建的仅指定 **msdyn_date** 和 **msdyn_duration** 的时间条目将在午夜开始。 **msdyn_start** 和 **msdyn_end** 字段将相应更新。

#### <a name="time-entry-types"></a>时间条目类型

时间条目记录具有关联的类型，其定义关联的应用程序的提交流中的行为。

|标签 | 值|
|-----|-----|
|在休息时   |192,355,000|
|旅行 | 192,355,001|
|加班   | 192,354,320|
|工作   | 192,350,000|
|休假    | 192,350,001|
|休假   | 192,350,002|


## <a name="customize-the-weekly-time-entry-control"></a><a name="customize"></a>自定义每周时间条目控件
开发人员可以向其他实体添加其他字段和查找，并实施自定义业务规则以支持其业务场景。

### <a name="add-custom-fields-with-lookups-to-other-entities"></a>向其他实体添加具有查找的自定义字段
向周时间条目网格添加自定义字段主要分三步。

1. 将自定义字段添加到 **快速创建** 对话框。
2. 配置网格以显示该自定义字段。
3. 根据需要将自定义字段添加到 **行编辑** 或 **时间条目编辑** 页面。

确保新字段在 **行编辑** 或 **时间条目编辑** 页面上具有所需的验证。 作为此任务的一部分，根据时间条目的状态锁定字段。

当您将自定义字段添加到 **时间条目** 网格，然后直接在网格中创建时间条目时，这些条目的自定义字段会自动设置，以与行匹配。 

### <a name="add-the-custom-field-to-the-quick-create-dialog-box"></a>将自定义字段添加到快速创建对话框
将自定义字段添加到 **快速创建：创建时间条目** 对话框。 然后，用户在通过选择 **新建** 添加时间条目时可以输入值。

### <a name="configure-the-grid-to-show-the-custom-field"></a>配置网格以显示该自定义字段
可通过两种方法向 **周时间条目** 网格添加自定义字段。

- 自定义 **我的周时间条目** 视图，并向其添加自定义字段。 可指定自定义字段在网格中的位置和大小，方法是在视图中编辑这些属性。
- 创建新的自定义时间条目视图，然后将其设置为默认视图。 除了您希望网格包括的列，此视图中还应包含 **说明** 和 **外部注释** 字段。 可指定网格的位置、大小和默认排序顺序，方法是在视图中编辑这些属性。 接下来，配置此视图的自定义控件，使其成为 **时间条目网格** 控件。 将此控件添加到视图，然后选择将其用于 **Web**、**手机** 和 **平板电脑**。 接下来，配置 **周时间条目** 网格的参数。 将 **开始日期** 字段设置为 **msdyn\_date**，将 **持续时间** 字段设置为 **msdyn\_duration**，将 **状态** 字段设置为 **msdyn\_entrystatus**。 **只读状态列表** 字段将设置为 **192350002（已批准）**、**192350003（已提交）** 或 **192350004（已请求撤消）**。

### <a name="add-the-custom-field-to-the-appropriate-edit-page"></a>将自定义字段添加到相应编辑页面
用于编辑时间条目或一行时间条目的页面可以在 **窗体** 下找到。 网格中的 **编辑条目** 按钮打开 **编辑条目** 页面，**编辑行** 按钮打开 **行编辑** 页面。 您可以编辑这些页面，让它们包含自定义字段。

两个选项都会删除对 **项目** 和 **项目任务** 实体的一些自带筛选，因此将显示这些实体的所有查找视图。 只有相关查找视图才会默认显示。

必须确定自定义字段的相应页面。 最可能的是，如果将字段添加到了网格，则应进入对适用于整行时间条目的字段使用的 **行编辑** 页面。 如果自定义字段在每一天的行中都有唯一值（例如，如果它是结束时间的自定义字段），它应该进入 **时间条目编辑** 页面。

要将自定义字段添加到页面，将一个 **字段** 元素拖到页中的相应位置，然后设置其属性。

### <a name="add-new-option-set-values"></a>添加新选项集值
要将选项集值添加到现成字段，请执行以下步骤。

1. 打开该字段的编辑页，然后在 **类型** 下选择该选项集旁边的 **编辑**。
2. 添加一个采用自定义标签和颜色的选项。 如果要添加新的时间条目状态，则自带字段将命名为 **条目状态**。

### <a name="designate-a-new-time-entry-status-as-read-only"></a>将新时间条目状态指定为只读
若要将新时间条目状态指定为只读，请将新时间条目值添加到 **只读状态列表** 属性中。 请务必添加数字，而不是标签。 现在将对具有新状态的行锁定时间条目的可编辑部分。 要为不同的 **时间条目** 视图的 **Read-only Status List** 属性进行不同的设置，在视图的 **自定义控件** 部分添加 **时间条目** 网格，并根据需要配置参数。

接下来，添加业务规则以锁定 **行编辑** 和 **时间条目编辑** 页面中的所有字段。 要访问这些页面的业务规则，打开每个页面的窗体编辑器，然后选择 **业务规则**。 可向现有规则中的条件添加新状态，也可以为新状态添加新业务规则。

### <a name="add-custom-validation-rules"></a>添加自定义验证规则
您可以为 **周时间条目** 网格体验添加两种类型的验证规则：

- 适用于页面的客户端业务规则
- 应用于所有时间条目更新的服务器端插件验证

#### <a name="client-side-business-rules"></a>客户端业务规则
可使用业务规则锁定和解锁字段，在字段中输入默认值，以及定义仅需要来自当前时间条目记录的信息的验证。 要访问页面的业务规则，打开窗体编辑器，然后选择 **业务规则**。 然后可以编辑现有业务规则或添加新的业务规则。

#### <a name="server-side-plug-in-validations"></a>服务器端插件验证
应该将插件验证用于需要的上下文比单个时间条目记录中的可用上下文多的任何验证。 您还应该将它们用于要在网格中的内联更新上运行的任何验证。 要完成验证，在 **时间条目** 实体中创建一个自定义插件。

### <a name="limits"></a>限额
目前，**时间条目** 网格的大小限制为 500 行。 如果超过 500 行，将不会显示多余的行。 无法增加此大小限制。

### <a name="copying-time-entries"></a>复制时间条目
使用视图 **复制时间条目列** 定义时间输入期间要复制的字段列表。 **日期** 和 **持续时间** 是必填字段，不应从视图中删除。
